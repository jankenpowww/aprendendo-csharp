C# a02 - Primeiro Programa no Padrão C# .NET

O que é o .NET Framework?

É a plataforma de desenvolvimento em C# criado e mantido pela Microsoft de forma totalmente única. Desenvolver programas C# no padrão .NET é desenvolver programas para além da família de sistemas operacionais do Windows. Você pode notar que essa é mesma ideia de outras linguagens semelhantes ao C#, como o Java, que utiliza da JVM para compilar e rodar programas escritos nessa linguagem em múltiplas plataformas.


Dando nome aos bois

Para entender melhor a estrutura apresentada por último na página anterior, iremos destrinchar e descobrir o que cada diretiva, palavras e estruturas aqui significam...


using System;
namespace a01;
class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
    }
}

Começemos pela linha using System;. Utilizamos a diretiva 'using' para toda vez que queremos importar uma biblioteca para o nosso projeto. As bibliotecas, ou como podem ser conhecidas como 'namespaces', podem ser instaladas através do Gerenciador de Pacotes do C#: o NuGet.

A biblioteca/namespace System é uma biblioteca/namespace padrão da linguagem que provê acesso a uma infinidade de funções referentes a manipulação de componentes fundamentais que são utilizados com bastante frequência durante o desenvolvimento de um programa em C#. À fim de um melhor entendimento, é através do namespace System que conseguimos acessar e utilizar o método Console.WriteLine(), que escreve um conteúdo de texto passado como string no terminal, por exemplo.

Note que, como mencionado anteriormente, a declaração de System propriamente dito pode ser desconsiderado atualmente devido atualizações da linguagem (visto que é alertado diretamente no editor de código do VSCode).

...

A linha seguinte, namespace a01, segue o mesmo conceito. 'a01', que é nome da pasta onde estão os arquivos-fonte do projeto, é utilizado como 'namespace' para guardar e ser utilizado para acessar as classes e métodos do arquivo-fonte onde eles foram declarados (?). Isso significa que, através de a01, podemos acessar a classe Program e de Program, acessar o método Main, como está descrito no código fonte acima. Em resumo, namespace guardam e organizam classes e métodos dentro de um nome especifico e referente ao arquivo fonte em que estão declarados.


static void Main(string[] args){}

Um método de modificador estático, ou static, referem-se aos métodos/classes/propriedades/etc. que são chamados e não são associados propriamente dito a uma instância de classe. Isso significa que, a chamada de um método estático pode ser feita apenas chamando o método em si, já que este tipo de método não é associado a uma classe propriamente dita. Isso signfica que métodos estáticos podem ser chamados independentemente de possuírem classes (vide código-fonte disponível neste diretório).

args é passado como parâmetro como um array de strings. Essa array de String diz respeito a cada versão de execução do programa que é organizada dentro desses índices e que podem ser acessados, em uma explicação muito breve.

De qualquer jeito, é necessário entender que métodos estáticos são métodos independente de classes. E é de um método estático que todas as outras classes e métodos organizados em outros arquivos são instanciados e executados dentro, uma vez que se faz a compilação do código.
