C# a17 - Array / Vetor

O que é?

Um vetor é como se chama o conjunto de variáveis organizada por índices de 0 até certo número de elementos, que armazenam valores. No C# e em linguagens de tipagem forte, os vetores reúnem um conjunto de elementos de um mesmo tipo, ao contrário de linguagens como JavaScript, que permite reunir strings, booleanos e objetos inteiros em uma única Array.

Mais uma vez: no C#, os vetores ou arrays são definidos com um tipo existente, primitivo ou não, referente à cada elemento a ser armazenado. Observe como criamos uma array unidimensional no C#:


//Arrays no C# - armazenam uma coleção de valores de tamanho dinâmico
//e de tipo definido.
int[] arrayNumeros = {143, 22, 354, 432, 43};

Podemos retornar o comprimento de um vetor acessando sua propriedade de classe 'Length'.


...

int[] arrayNumeros = {143, 22, 354, 432, 43};

Console.Write($"Há {arrayNumeros.Length} elementos dentro do vetor")

E assim como com strings, podemos retornar um valor com base em um índice existente dentro de um Array...


...

int[] arrayNumeros = {143, 22, 354, 432, 43};

//Retorna 432.
Console.WriteLine("O valor presente no índice 3 é {0}.", arrayNumeros[3]);

... E também é possível substituir um valor de índice ou adicionar um novo índice à Array de forma muito simples. Veja:


...

int[] arrayNumeros = {143, 22, 354, 432, 43};

//Substituíndo o valor do índice 1.
arrayNumeros[1] = 7;

//A array completa será composta por...
//143, 7, 354, 432, 43

E atenção: diferente do que é visto em outras linguagens, a forma com que declaramos uma array em C# aqui foi definir uma array de tamanho fixo. Isso significa que, apesar do momento de declaração de arrayNumeros definirmos o seu comprimento total de acordo com os elementos presentes nele, isto é, um array de inteiro com 5 elementos/de comprimento, não significa que podemos, de primeira maneira, adicionar mais um elemento à array. Isso significa que não existe o índice de número 5 em diante na nossa array, justamente por ser um array de tamanho fixo, e não dinâmico.

Para entender melhor como arrays de tamanho fixo funcionam na linguagem, vamos considerar esse pedaço de código, onde declaramos uma array e seu tamanho fixo de forma mais objetiva, ao contrário de defini-la populando com elementos diretamente.


int[] numeros = new int[3];

Acima, declaramos um array de inteiro de tamanho 3. E sim, diferente das outras sintaxes de outras linguagens da qual estamos acostumados, essa forma de declarar, ou melhor, instanciar uma array, não indica que '3' é um elemento que faz parte da array. Note que não há diferença entre a forma anterior e esta de se declarar/instanciar arrays, todas sem exceção continuam sendo arrays de tamanho fixo. O que muda de um para o outro é que, desta forma, o tamanho da array é definida explicitamente, enquanto ao dos códigos anteriores, a definição de tamanho se dá pelo tanto de elementos presentes no momento da instanciação.

De toda forma, numeros é um array de comprimento 3, onde os espaços vão do índice 0 até, sendo ele um array fixo.

Se tentarmos adicionar um elemento ao índice 3, da qual está além do limite de comprimento da Array, o resultado não poderia ser outro, se não um erro:


Unhandled exception. System.IndexOutOfRangeException: Index was outside the bounds of the array.
