C# a08 - Lendo Valores do Teclado

Em aplicações com C# baseadas no console, é imprescindível a entrada e saída de informações. No mais, a entrada de dados é tão importante quanto a saída de dados, já que uma coisa se complementa a outra quando estamos desenvolvendo pequenos programas desse tipo.

Como visto anteriormente, utilizamos Console.ReadLine() para toda vez que queremos com que o usuário entre com alguma informação no console. Dependendo do tipo de informação, vimos que é necessário fazer conversões de tipos, em especial quando o valor de entrada precisam ser números, vide exemplo abaixo:


//Comparando valores e testando se são valores iguais ou não.

Console.Write("Digite um valor: ");
int n1 = int.Parse((Console.ReadLine()));

Console.Write("Digite outro valor: ");
int n2 = int.Parse((Console.ReadLine()));

string resultado = n1 == n2 ? "São valores iguais." : "São valores diferentes.";

Console.WriteLine(resultado);

...

Mas para além do método de captura de valores convencional, temos também outro comando que vez ou outra parece sendo sugerido que é o Console.ReadKey(). Este método é utilizado para registrar o valor dos botões pressionados no teclado. E, não, não basta atribuir o valor numa string e tentar exibir na tela. Há um "tipo primitivo", entre muitas aspas, no C# especifico para este tipo de ocorrência. Este tipo que e considerado na verdade uma estrutura do C# para lidar com botões pressionados no teclado, de maneira mais básica.


Console.WriteLine("Aperte qualquer botão...");

//ConsoleKeyInfo é a estrutura que define variáveis que "recebem valores 
//pressionados no teclado". ReadKey() pode ser acessado novamente em outras 
//propriedades durante a atribuíção de valores, que agora, é
//acessado pelo valor atribuído.

//Há muitas formas de capturar um valor do teclado, vide código-fonte do a08cs3.

ConsoleKeyInfo botaoPressionado = Console.ReadKey();

Console.WriteLine("Botão pressionado: " + botaoPressionado.Key.toString());
