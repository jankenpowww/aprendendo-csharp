C# a03 - Variáveis em C#

... e o que é linguagem de tipagem forte.

Linguagens fortemente tipadas como Java e até mesmo o C# se enquadram nessa definição de "linguagem de tipagem forte". E o que isso significa, afinal? De forma bastante resumida e nada técnica, diga-se de passagem as linguagens tipadas são aquela que exigem que tipos primitivos sejam fixos para suas variáveis: isso significa que, se variável x é do tipo inteiro, seu valor deve ser um inteiro, independentemente do valor que entra ou é modificado por outro.

Seguindo a lógica, podemos interpretar que linguagens fracamente tipadas possuem uma filosofia diferente, e até mais liberal nesse sentido, à respeito de suas variáveis. PHP, Python e outras linguagens onde variáveis são dinâmicas podem receber uma string até certo momento da execução de um código e, em outra, receber um valor inteiro. Essa é uma das grandes características divergentes entre linguagens interpretadas e linguagens compiladas, inclusive. JavaScript, que é uma linguagem de tipagem fraca, permite que variáveis recebam não só strings e valores numéricos literais, mas como também funções com blocos de código inteiros, coisa que em linguagens de tipagem forte já não permitem.


Variáveis em C#

No C#, a síntaxe de declaração de variável segue primeiramente declarando o tipo primitivo e em seguida o nome identificador da variável. Variáveis podem ser declaradas sem receber nenhum valor aparente, mas é bom se atentar com erros causados durante execuções de código por conta das variáveis não serem inicializadas.

O programinha a seguir pede nome e idade do usuário. Sabendo que 'nome' é um tipo de dado constituído por caracteres e 'idade' é um valor numérico inteiro, temos as nossas primeiras variáveis criadas e seus tipos definidos!

Ao capturar, vamos devolver essa informação no terminal em um texto bonito que concatena variável e texto. Observe:


namespace a03;
class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Qual é o seu nome?");
        string nome = Console.ReadLine();

        Console.WriteLine(nome + ", quantos anos você tem?")
        int idade = Console.ReadLine();

        Console.WriteLine($"\nSeu nome é {nome} e você tem {idade} anos de idade.")
    }
}

E rodando o código acima, temos...


... error CS0029: Não é possível converter implicitamente tipo "string" 
em "int" ...

Um erro de código. E exatamente na variável que vai receber o valor referente a idade (já que é o único de tipo de variável setada como int). Porquê?

Por que quando utilizamos o método de entrada de informações no C#, ou Console.ReadLine() pros mais íntimos, precisamos entender que toda informação digitada pelo usuário no terminal é um valor String por padrão. Uma das soluções possíveis seria alterar o tipo primitivo da variável em questão para o mesmo tipo de nome, que é string.

Mas e se partirmos do pressuposto de que uma variável de tipo numérico PRECISA ser de tipo numérico, uma vez que podemos lidar com comparações de valores e que isso só é possível quando uma informação de uma variável é um número, por exemplo?

A alternativa correta seria converter o tipo de dado recebido para o tipo de dado apropriado. Neste caso, a variável 'idade', que recebe um valor inteiro (int), está recebendo um valor de cadeia de caracteres (string). Faremos uma conversão de valores de 'string' para 'int'.


...
        int idade = int.Parse(Console.ReadLine());
...

Basicamente passamos o método 'Console.ReadLine()' dentro de outro método como parâmetro, o int.Parse(). 'int', além de tipo primitivo, também é nome de uma estrutura do C# que pode ser utilizada para acessar métodos referente ao valor de tipo primitivo de numeros inteiros, um desses métodos é justamente esse de converter um valor de tipo tal para um valor numérico de tipo inteiro, desde que o valor corresponda a um valor numérico que possua parte inteira.

Perceba como as variáveis aqui devem ser estritamente respeitadas quanto ao seu tipo primitivo. Essa é uma demonstração objetiva de como linguagens de tipagem forte funcionam.


Os tipos de variáveis em C#

Confira a lista com esses e outros tipos primitivos interessantes dentro do C# que podem ser utilizadas:

Tipo Descrição
int Recebe valores numéricos inteiros.
string Recebe valores de cadeia de caracteres. Em resumo: texto, número, símbolos, espaços e o mais importante: tudo o que está entre aspas.
char Recebe, em aspas símples, um valor/digito ou caracter único.
float Recebe um valor decimal com uma precisão de 32 bits.
double Recebe um valor decimal com uma precisão de 64 bits.
decimal Recebe um valor decimal com uma precisão de 128 bits.
bool Recebem dois valores booleanos possíveis: 'true' ou 'false'.
var O mais próximo que temos de "variável dinâmica" no C#. Seu tipo é definido de uma vez por todas de acordo com o que ele receber primeiro. Se uma variável do tipo 'var' receber string, ele só pode receber string, e o mesmo vale com informações de outros tipos.

Vale lembrar que o tipo 'int', assim como os tipos de valores decimais, também se subdividem em outros tipos de inteiro onde se especifica seu comprimento em bits. 'int' antecede, em termos de comportar valores em bits do menor para o maior, o 'uint', 'long', 'ulong', etc. Todos esses são ramificações de tipo inteiro dentro do C#. O 'int', que é o mais comumente utilizado, suporta uma informações de número inteiro de comprimento de 32 bits.
