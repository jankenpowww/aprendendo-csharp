C# a01 - Preparando o ambiente.

Por se tratar de uma linguagem de programação mantida pela Microsoft, versões do C# podem estar disponíveis dentro da pasta Window > Microsoft.NET. Isso significa que é possível desenvolver programas em C# com a versão atual instaladas em um sistema operacional do Windows.

Mas para se manter atualizado com as últimas versões lançadas da linguagem, ainda é necessário fazer suas instalações. Em https://dotnet.microsoft.com/downloads, é possível encontrar essas últimas versões do C# e do .NET Framework necessárias para o desenvolvimento. A versão utilizada do C# aqui é a versão 7.0 do .NET Framework SDK.

Para este começo, a IDE utilizada será o Visual Studio Code. Extensão do C# para a linguagem também deve ser instalada.

Importante! Note que, apesar do C# estar instalado no Windows por padrão, o seu compilador, o csc.exe, não pode ser executado por conta do Windows não reconhecê-lo. Para resolver isso, basta adicionar o caminho da pasta onde se encontra o csc.exe no Path do Windows, em variáveis de ambiente. Para isso, você deve localizar o caminho da pasta do executável (não o caminho para o executável), editar o Path dentro das variáveis de ambiente para todo o sistema, e adicionar um novo caminho que é o caminho do executável e confirmar tudo.

As mudanças só ocorrem depois de reiniciar o computador. Depois de reiniciar, tente executar o comando csc no prompt de comando. A ideia aqui é que a mensagem de que "o Windows não reconhece como comando interno ..." não apareça dessa vez. Se não, como esperado, é sinal de que compilar os programas escritos em C# é totalmente possível via terminal! Caso contrário, verifique se o caminho especificado é da pasta onde se encontra o compilador, e não um caminho que inclua o executável em si.


Programando em C#

Tudo devidamente pronto, vamos conhecer finalmente os primeiros processos de programação em C#. Mas para isso, temos de entender como é a estrutura principal de um programa em C# e alguns primeiros comandos. Observe...

No terminal, já dentro do VSCode e dentro da pasta do projeto, podemos inicializar um projeto em C# com o seguinte comando:


dotnet new console

Isso vai gerar no explorador as pastas e arquivos do projeto com a versão mais recente instalada do C#. O projeto de nome "Program.cs" é o que importa aqui agora. Ao abrir, temos de cara a seguinte "estrutura" no nosso código-fonte:


// See https://aka.ms/new-console-template for more information
Console.WriteLine("Hello, World!");

Isso mesmo: nenhuma estrutura! E isso porque as versões novas do C#/.NET à partir de 6.0 contam com algo chamado de "instruções de nível superior". Isso significa que os novos modelos de projeto não necessitam daquela estrutura de uma classe principal para rodar um programa.

Para rodar o código gerado acima precisamos garantir que ainda estamos na pasta do projeto da qual geramos a estrutura de arquivos e pastas com o comando anterior. E no terminal, digitamos...


dotnet run

E ele retornará "Hello, World!" no console no mesmo instante.


Porém, contudo, todavia...

Embora estejamos utilizando e lidando com features novas das versões mais recentes do C#, por questão de aprendizado, vamos aprender e entender a escrever em C# utilizando da sua estrutura, e compreender de uma vez por todas como funciona essa linguagem que é primariamente orientada a objeto.

Podemos gerar um projeto com a estrutura convencional sem nenhum problema utilizando dessa vez a flag --use-program-main quando inicializarmos um novo projeto.


dotnet new console --use-program-main

E abrindo o Program.cs no editor, temos finalmente nossa estrutura!


namespace a01;
class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
    }
}

É interessante notar que o uso da diretiva using aqui, considerando as novas versões em comparação com as estruturas padrões em versões anteriores, não são incluídos mais e são desconsiderados pelo próprio editor.
